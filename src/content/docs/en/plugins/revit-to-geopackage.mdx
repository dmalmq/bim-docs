This plugin exports 2D Revit views (floor plans, site plans) into GeoPackage
(`.gpkg`) or Shapefile (`.shp`) format so they can be used in QGIS, ArcGIS, or
any GIS platform.

## Prerequisites

- The Revit model must have **shared coordinates** set correctly.
  See [New Project Setup](/en/revit/new-project-setup/).
- The plugin add-in (`.addin`) must be installed. Check with your BIM lead if
  you do not see it in the **Add-Ins** tab.

## Export Workflow

<Steps>
1. Open the **view** you want to export (e.g. a floor plan at Level 1).
2. Go to **Add-Ins → Export to GeoPackage**.
3. Choose the **output format** — GeoPackage is preferred unless the recipient
   specifically needs Shapefile.
4. Select the **coordinate reference system** — this should match what was set
   in the project setup (e.g. EPSG:6677).
5. Choose which **categories** to include (Walls, Doors, Rooms, etc.).
6. Click **Export** and choose a save location.
</Steps>

<Aside type="tip">
  Exporting Rooms will include their area and name as attribute data in the
  GeoPackage — useful for IMDF conversion later.
</Aside>

## Output Structure

The exported GeoPackage contains one layer per Revit category:

| Layer         | Geometry   | Key Attributes              |
| ------------- | ---------- | --------------------------- |
| `walls`       | LineString | Type, Width, Level          |
| `doors`       | Point      | Type, Width, Height, Room   |
| `rooms`       | Polygon    | Name, Number, Area, Level   |
| `furniture`   | Point      | Family, Type                |

## Troubleshooting

**Geometry is offset from expected location** — The shared coordinates in the
Revit model are likely wrong. Re-check the survey point values against the
surveyor's data.

**Empty layers** — Make sure the categories you selected actually have visible
elements in the exported view. Hidden elements are not exported.
